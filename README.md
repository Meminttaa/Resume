## 🎖️ VIP-Модуль: Программа допуска к экзамену
> **Обязательное условие:** Автоматическое биометрическое распознавание студента на фотографии для верификации личности.

Программа сканирует загруженное изображение, обнаруживает лица, сопоставляет их с локальной базой данных («слепков») и мгновенно выводит имя идентифицированного человека прямо на экран. Все десктопные интерфейсы приложений выполнены в кастомном стиле **Ruby Crimson Noir** *(глубокие бордовые оттенки `#1a080a`, контрастный светящийся лог-текст `#ffb3ba` и рубиновые акценты)*.

### 🧠 Механика работы ИИ-распознавания:
1. **Экстракция дескрипторов:** Модуль использует глубокие свёрточные нейросети через библиотеку `face_recognition` (надстройка над `dlib` и `OpenCV`).
2. **Векторный анализ:** Каждое лицо кодируется в **128-мерный вектор биометрических параметров** (эмбеддинг).
3. **Метрическое сопоставление:** Сравнение тестового кадра с базой данных `dataset/` происходит по расчету евклидова расстояния между векторами с жестким порогом чувствительности `tolerance=0.55`.
4. **Асинхронный поток:** Чтобы тяжелый инференс модели не замораживал GUI, процесс распознавания вынесен в отдельный поток `threading.Thread`.

> [!WARNING]
> ### 🔒 Важная сноска по датасету (Крипто-режим локализации)
> Текущий ИИ-модуль распознавания содержит в себе **крайне ограниченную базу лиц**. В процессе сбора биометрических данных подавляющее большинство одногруппников проявило паранойю, отказалось делиться своими фотографиями и предпочло остаться в режиме строжайшего цифрового инкогнито. Модель обучена в условиях дефицита данных, но работает с филигранной точностью на доступной выборке.

💡 ПАСХАЛКА: В утилиту встроен скрытый Aim-тренер («Поймай баг!») на базе Tkinter.
Разгрузите процессор, покликайте по убегающим квадратам, если ИИ слишком долго ищет лица!

<details>
<summary>📂 <b>01.01 & 01.03 Фильтрация изображений (Алгоритмы OpenCV)</b></summary>
<p></p>

* **Размерный фильтр:** Проводит валидацию разрешения матрицы через кортеж `img.shape`. Картинки меньше `150x150px` бракуются для защиты ИИ от пиксельного шума.
* **Цветовой фильтр:** Функция разбивает изображение на цветовые каналы `cv2.split(img)`. Через `np.array_equal(b, g)` проверяется идентичность матриц. Если каналы равны — изображение классифицируется как `⚪ Черно-белое`, иначе — `🔴 Цветное RGB/BGR`.
* **Шаблонный фильтр:** Загружает ультра-легкий каскад Хаара `haarcascade_frontalface_default.xml` для обнаружения геометрии лиц через `detectMultiScale` (шаг сканирования `scaleFactor=1.1`, минимальные соседи `minNeighbors=5`).

<details>
<summary>📂 <b>01.02 Изоляция GUI в Модульных тестах (Unittest)</b></summary>
<p></p>

Главная сложность тестирования Desktop-приложений — падение тестов в Docker из-за отсутствия графического дисплея (X-сервера). Архитектура тестов решает это через **Deep Mocking**:
```python
import sys
from unittest.mock import MagicMock

# Подмена графического движка до импорта основного кода
sys.modules['_tkinter'] = MagicMock()
sys.modules['tkinter'] = MagicMock()
sys.modules['face_recognition'] = MagicMock()
sys.modules['cv2'] = MagicMock()
Тесты перехватывают системный кэш sys.modules, подменяя графические и тяжелые ИИ-библиотеки на заглушки MagicMock. Это позволяет мгновенно верифицировать логику функций, создание папок и логирование, обеспечивая скорость выполнения тестов в 0.01 сек без отрисовки окон.

Развернут оптимизированный Dockerfile для запуска тестов и ИИ-скриптов в изолированном Linux-окружении:

Базовый слой: python:3.11-slim минимизирует вес итогового образа.

Системные зависимости: Локально компилируются необходимые утилиты сборки (cmake, build-essential) и библиотеки рендеринга графики (libgl1, libglib2.0-0, tk-dev).

Кэширование слоев: Зависимости ставятся без сохранения временных файлов через параметр --no-cache-dir.

Автоматизация запуска: Контейнер настроен на чистый автономный прогон тест-пакета через инструкцию CMD ["python", "test_recognize.py"].

📝 Математическая модель NLP (Классификатор текста)
Модуль предсказывает тональность по 4-м независимым классам (Позитив, Нейтральный, Смешанный, Негатив):

Векторизация: Текст переводится в матрицу частотности токенов с помощью CountVectorizer.

Алгоритм: Применяется Мультиномиальный Наивный Байес (MultinomialNB), высчитывающий параллельное распределение вероятностей принадлежности фразы к классу на основе теоремы Байеса.

Дообучение "на лету": Реализована фича динамического сохранения весов. При неверной классификации интерфейс позволяет отправить фразу в БД бэкенда, мгновенно вызывая метод partial_fit() и перезаписывая дампы моделей через joblib.

Архитектурный паттерн разделения обязанностей:

Django 5.2 (Панель управления и ORM): Отвечает за рендеринг сайта-резюме (views.index_page), менеджмент администраторов и строгое описание реляционных моделей в PostgreSQL (models.py).

FastAPI (Высоконагруженный ИИ-микросервис): Берет на себя асинхронные эндпоинты (ai/analyze-text/, ai/recognize-faces/). Позволяет обрабатывать тысячи конкурентных запросов на инференс нейросетей, не блокируя основной поток ввода-вывода веб-сайта.
