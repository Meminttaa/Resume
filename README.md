## 🛑 Программа распознавания лиц (Допуск к экзамену)

> **Задание:** Написать программу, которая автоматически распознает студента на фотографии для подтверждения личности и выводит его имя. Приложение может быть одно на группу.

Интерфейс программы оформлен в темной палитре **Ruby Crimson Noir** *(глубокий бордовый фон `#1a080a`, лог-терминал `#120405` с розовым текстом `#ffb3ba` и рубиновые акценты на кнопках)*. Чтобы интерфейс не «зависал» во время анализа кадра, сам процесс распознавания вынесен в отдельный поток `threading.Thread`.

### 🛠️ Как это устроено внутри кода:
1. **Поиск и разбор лица:** Приложение использует библиотеку `face_recognition` (работает поверх `dlib` и `OpenCV`). Она находит лицо на фото и кодирует его параметры в цифровой вектор (128 чисел).
2. **Сравнение со слепками:** Полученный вектор сравнивается с уже готовыми изображениями из папки `dataset/` методом расчета евклидова расстояния. 
3. **Порог точности:** В коде выставлен жесткий порог `tolerance=0.55`. Если отличий больше, программа человека не пропустит (защита от случайных совпадений).

> [!WARNING]
> ### 🔒 Сноска касательно базы данных (датасета)
> Модуль распознавания сейчас содержит **крайне мало лиц**. В процессе сбора фотографий для базы данных большинство одногруппников наотрез отказались делиться своими снимками и предпочли остаться инкогнито. Программа работает точно, но только на тех немногих, кто согласился поучаствовать в тестировании.

💡 К слову: В интерфейс добавлена скрытая мини-игра (Aim-тренер) на Tkinter.
Можно покликать по убегающим квадратам и немного отвлечься, пока ИИ обрабатывает тяжелые файлы.

<summary>📦 <b>01.01 & 01.03 — Логика фильтрации картинок (OpenCV)</b></summary>
<p></p>

* **По размеру:** Проверяем разрешение через свойства матрицы `img.shape`. Картинки меньше `150x150px` сразу отсекаются, так как на них сложно точно разобрать детали.
* **По цвету:** Делим картинку на каналы `cv2.split(img)` и сравниваем их между собой через `np.array_equal`. Если синий, зеленый и красный каналы полностью идентичны — это черно-белое изображение, если есть отличия — цветное.
* **По шаблону:** Используем встроенный в OpenCV стандартный файл `haarcascade_frontalface_default.xml`. Метод `detectMultiScale` ищет на фото специфические контуры и перепады освещения, характерные для человеческого лица.

<details>
<summary>📦 <b>01.02 — Как тестировать интерфейс в консоли (Unittest Mocking)</b></summary>
<p></p>

Главная проблема при тестировании оконных приложений в Docker — отсутствие графической оболочки (X-сервера), из-за чего тесты падают с ошибкой. Чтобы обойти это, в коде используется полная изоляция среды:
```python
import sys
from unittest.mock import MagicMock

# Подменяем модули до того, как их импортирует основной скрипт
sys.modules['_tkinter'] = MagicMock()
sys.modules['tkinter'] = MagicMock()
sys.modules['face_recognition'] = MagicMock()
sys.modules['cv2'] = MagicMock()
Скрипт перехватывает системный кэш sys.modules и подсовывает заглушки вместо реальных библиотек tkinter и cv2. В итоге тесты проверяют логику функций, работу с папками и логи за сотые доли секунды, даже не пытаясь открыть само окно программы.

Для изоляции проекта написан лаконичный Dockerfile:

За основу взят чистый python:3.11-slim, чтобы образ весил как можно меньше.

Внутри выполняются команды сборки компилятора (cmake, build-essential), а также ставятся системные пакеты для работы с графикой (libgl1, tk-dev).

Параметр --no-cache-dir при установке зависимостей не сохраняет временные файлы pip, экономя место.

Контейнер по умолчанию настроен на автоматическую проверку проекта: инструкция CMD сразу запускает файл с тестами test_recognize.py.

📝 Как работает определение тональности:
Модель делит текст на 4 категории (Позитив, Нейтральный, Смешанный, Негатив).

Сначала текст переводится в числовой массив частоты употребления слов через CountVectorizer.

Матрица передается классификатору Мультиномиального Наивного Байеса (MultinomialNB), который вычисляет вероятность принадлежности фразы к каждому классу.

Добавлена функция «дообучения на лету»: если ИИ ошибся, через интерфейс можно отправить правильный вариант в базу бэкенда, вызвав метод partial_fit(), и обновить веса модели в файле .pkl с помощью библиотеки joblib.

В этой работе применено разделение серверной логики:

Django 5.2: Используется для классического вывода страниц сайта-резюме, удобной работы с базой данных PostgreSQL через стандартные модели Django ORM (models.py) и администрирования.

FastAPI: Выделен под отдельные асинхронные эндпоинты (ai/analyze-text/ и ai/recognize-faces/). Он быстро принимает запросы, передает их ИИ-скриптам на обработку и возвращает результат в формате JSON, не блокируя работу основного сайта.
